# Time-Blocking Framework

This file defines the rules for how Long's day is structured. The assistant must respect these blocks when proposing time slots.

---

## Daily Block Template

```
07:00 – 09:00  │ SIDE PROJECT / DEEP PERSONAL WORK
               │ Protected time for side projects or personal growth tasks.
               │ No meetings. No work emails.

09:00 – 12:00  │ DEEP WORK (Work)
               │ High-focus work tasks: writing, coding, strategic thinking.
               │ Meetings only if unavoidable.

12:00 – 13:00  │ LUNCH BREAK
               │ No tasks scheduled.

13:00 – 15:00  │ COLLABORATIVE / MEETING BLOCK (Work)
               │ Preferred time for meetings, calls, and syncs.

15:00 – 17:00  │ ADMIN & EXECUTION (Work)
               │ Emails, follow-ups, planning, smaller work tasks.

17:00 – 18:00  │ WRAP-UP (Work)
               │ Review, handoffs, plan for tomorrow.

18:00 – 19:00  │ PERSONAL TRANSITION
               │ Exercise, commute, wind-down. No task scheduling.

19:00 – 21:00  │ PERSONAL / SIDE PROJECT (Evening)
               │ Errands, personal admin, side project overflow.

21:00+         │ OFF
               │ No tasks to be scheduled past this time.
```

---

## Scheduling Rules

1. **Match task type to block type.** A work task should never be placed in a personal block, and vice versa.
2. **Respect existing events.** Always check the calendar for conflicts before proposing a slot.
3. **Add buffer after meetings.** Leave a 15-minute buffer after any meeting before scheduling focused work.
4. **Batch similar tasks.** If there is already an errand or admin task nearby, try to batch the new task in the same block.
5. **Don't overfill a block.** If a block already has 90%+ capacity, propose the next available block instead.
6. **P1 tasks take priority.** If a P1 task arrives, find the earliest available deep work slot today, even if it means rearranging.
7. **P4 tasks don't get scheduled.** Log them in Notion without placing them on the calendar.

---

## Weekend Rules

- Saturday morning (9:00–12:00): Side projects or personal deep work
- Saturday afternoon: Personal / errands / social
- Sunday: Reserved for rest and weekly review — avoid scheduling unless P1
